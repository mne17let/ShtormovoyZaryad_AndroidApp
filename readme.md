Сервер лежит на http://mne17let.beget.tech

Чтобы зарегистрировать пользоватея нужно отправить post-запрос на http://mne17let.beget.tech/mobile/registration/ с словарем, состоящим из 4 полей:
    'password'='new_password'
    'again_pass'='new_password'
    'mail'='your_email'
    'username'='new_login'
При корректном вводе данных сервер зарегистрирует нового пользователя и вернет ответ со статусом 200 (Ok) и соощением 'user is successfully created'.
При неправильном вводе данных сервер вернет ответ с разными кодами ошибок (для удобной обработки) и некоторым сообщением.
    Пароли не совпадают: content:'bad password', status_code:406
    Неверный mail: content:'bad email', status_code:407
    Слабый пароль (я поставил некотрую валидацию паролей, чтобы пароль не состоял из одной буквы, например): content:'bad password', status_code:408
    Cлабый login: content:'bad login', status_code:409
    login уже существует в системе: content:'user already exists', status_code:410
    Что-то пошло не так (если возникла некоторая ошибка при регистрации usera): content:'something went wrong', status_code:200
Идейно: После регистрации пользователя я не даю никаких токенов. Чтобы пользователь смог зайти под своим логином, его нужно перенаправить на страницу логина, где он сможет войти от своего имени.

Чтобы авторизовать пользователя нужно отправить get-запрос на http://mne17let.beget.tech/mobile/login/, после чего вернется некоторый куки-файл c полем name='csrftoken': value='some token'. Этот токен нужен для защиты от csrf-атак. Не вникайте что это, просто при следующих post-запросах нужно добавлять в словарь, который вы отправляете поле 'csrfmiddlewaretoken': 'some token'. (раньше он был в куках с названием 'csrftoken', а теперь в поле 'csrfmiddlewaretoken' в словаре пост запроса. После этого следует отправлять post-запрос c введенными пользователем данными ('username':'your login', 'password':'your password') на тот же адресс. Если аутентификация прошла не успешно, я верну ответ со статусом 405. Если же успешно, то я верну ответ со статусом 200 и в куки-файле добавится name='sessionid', value='some token'. этот куки файл нужно сохранить и при следующих запросах к серверу, последний будет проверять этот sessionid и знать, какой пользователь отправляет запрос (если sessionid юудет отстутствовать в куках запроса, то сервер будет думать, что вы неавторизованный пользователь и в теории при таких запросах нужно возвращать пользователя на страницу с авторизацией)

Чтобы разлогиниться нужно отправить get-запрос на http://mne17let.beget.tech/mobile/logout, тогда сервер удалит запись о том, кому принадлежит ключ sessionid и в теории должен вернуть пользователя на страницу с авторизацией.
